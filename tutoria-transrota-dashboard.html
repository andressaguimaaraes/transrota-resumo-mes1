Tópico 06
🔐 Segurança no GitHub
Usar arquivos git ignore para proteger chaves de API e dados sensíveis.
Fazer revisões periódicas de segurança no código. Recomendado usar as ferramentas de "review" do Cloud Code para identificar erros por prioridade (alta/média/baixa). Nunca subir chaves de API, tokens ou senhas no repositório.

Tópico 09
⚙️ Desempenho e Modelos de IA
Para resolver lentidão: avaliar migrar a hospedagem para Vercel.
Níveis de modelo de IA: Junior (tarefas simples), Sênior (uso geral), Especialista (alta complexidade e segurança — custo maior, mas necessário). Também avaliar revisar o uso do banco de dados como causa de timeout.

Tópico 11
🛠️ Boas Práticas de Desenvolvimento
Testar em ambiente local antes de subir para produção.
Usar GitHub para versionamento — permite voltar a commits anteriores se algo quebrar. Antes de novas integrações: montar um planejamento estruturado, decidindo entre ferramentas externas ou centralizar na própria plataforma.

Tópico 13
📄 PRD — Padrão para novos projetos
Documento de Requisitos de Produto para estruturar projetos antes de desenvolver.
O PRD contém: nome do projeto, problema, personas, solução, estilo visual e especificações técnicas. Reduz retrabalho e gera um arquivo Markdown (.md) para usar direto na plataforma de desenvolvimento ou em chats. Diogo vai enviar o modelo até 03/07.
▼ Ver mais
Tópico 05
🖥️ Plataforma Própria — Estabilidade
A plataforma interna (baseada em navegador) apresenta problemas de lentidão (time out).
Precisa de auditoria e refinamento do código. Possíveis causas: hospedagem inadequada, uso excessivo do banco de dados, código não otimizado. Solução a testar: migrar para Vercel + revisar queries de banco.

🗓️ Sessões Individuais
Haverá sessões personalizadas para analisar ferramentas de cada participante.
Exemplos: plataforma do Jonathas, Aurora da Maria Estela. Inclui brainstorming de melhorias. Agendamento direto pela plataforma Winner.ia (vinculada à agenda do Diogo). A equipe deve preencher o formulário de feedback da tutoria.

Tópico 12
🎨 Identidade Visual de Referência
É possível usar sites como referência para definir identidade visual própria.
Exemplo: padrão visual da Apple para tipografia e espaçamento. Usar como base para design próprio — sem cópia integral, sem problema jurídico. Define identidade, espaçamento, tipografia e hierarquia visual.
